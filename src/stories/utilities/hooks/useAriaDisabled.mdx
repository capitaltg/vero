# useAriaDisabled

Provides accessibility-friendly disabled behavior that keeps elements in the tab order while preventing interaction. Prefer this over the native `disabled` attribute so keyboard users can still focus and read the element.

## Parameters

  <thead>
    <tr>
      <th>Name</th>
      <th>Type</th>
      <th>Default</th>

        <code>false</code>
      </td>

## Returns

When `isDisabled` is `false`, returns an empty object. When `true`, returns `aria-disabled: true` plus event handlers that block interaction:

- Mouse: `onClick`, `onMouseDown`, `onMouseUp`
- Touch: `onTouchStart`, `onTouchEnd`
- Pointer: `onPointerDown`, `onPointerDownCapture`
- Keyboard: `onKeyDown`, `onKeyUp` — Tab and Escape are still allowed
- Form: `onInput`, `onSubmit`, `onReset`
- Clipboard: `onCopy`, `onCut`, `onPaste`
- Drag: `onDrag`, `onDragStart`, `onDragEnd`, `onDrop`

## Example

```tsx
import { useAriaDisabled } from '@capitaltg/vero';

function MyButton({ isDisabled, children }) {
  const disabledProps = useAriaDisabled({ isDisabled });
  return (
    <button
      className="aria-disabled:cursor-not-allowed aria-disabled:opacity-50"
      {...disabledProps}
    >
      {children}
    </button>
  );
}
```
